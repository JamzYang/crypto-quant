## 一、你目前已经覆盖了哪些能力？

结合三份文档，现在已经比较清晰的能力块主要有：

- **实战数学能力清单**  
  覆盖收益分布、概率直觉、尾部风险、相关性/回归、时间序列直觉、风险指标、资金管理等。  
  这是“算得明白、看得懂报表”的底层技能。

- **量化系统整体框架 & 工具栈认知**  
  在《路线图》中，数据层 → 策略层 → 回测层 → 执行层 → 风控层的结构已经很清楚。  

- **学习重心的比例意识**  
  已经明确：  
  - 量化理论 / 市场理解：60%  
  - Python + 数据分析 + 回测：30%  
  - 工程化系统：10%  

但这些更多是“**概念 + 数学 + 一点实作**”，还缺几块**偏“运营人/研究负责人”的能力清单**。

## 二、除了工程和代码，你关键缺的 6 大能力清单

下面这 6 份“清单”都是**偏思维、框架和流程**，很少涉及具体代码，更贴近“小型私募运营人”的角色。

### 1. 市场理解与交易框架清单

这块在路线图里有零散提到，但没有系统成清单。建议单独列：

- **市场结构与参与者**
- **币圈特有机制**：永续合约、资金费率、爆仓机制、强平规则
- **市场状态 / regime**：趋势、震荡、暴跌、减半/牛熊周期
- **交易成本与微观结构**：点差、深度、滑点、吃单/挂单、限价/市价
- **典型“别人输钱的方式”**：高杠杆梭哈、追高杀跌、FOMO/FUD 行为模式

这份清单的目标：  
让你在看任何策略时，都能回答：**“它是在什么市场结构下赚谁的钱？”**

> 建议文件名：`docs/guide/市场理解与交易框架清单.md`

---

### 2. 策略思维与研究方法清单（非代码）

现在的内容里，“策略原型”有，但**“怎么做研究”本身还没有清单化**。可以做一份：

- **策略类型地图**：趋势、均值回归、套利、做市、事件驱动、因子/统计套利
- **交易假设写法模板**  
  - 标的/周期  
  - 边际信息（信号）  
  - 逻辑：为什么会有 edge  
  - 主要风险与失效场景
- **实验设计与回测原则（比纯数学更实务）**  
  - 样本内 / 样本外划分  
  - 训练 / 验证 / 测试集  
  - Walk-forward / 滚动回测思路
- **鲁棒性检查清单**  
  - 参数敏感性测试  
  - 不同市场阶段测试  
  - 子样本稳定性
- **策略迭代流程**  
  - idea → 快速原型 → 初步回测 → 砍掉/改进 → 加入组合或淘汰

目标：形成一套**“研究方法 SOP”**，避免变成“凭感觉乱试策略”。

> 建议文件名：`docs/guide/策略研究方法与实验设计清单.md`

### 3. 风控与资金管理“制度化”清单（超越数学）

实战数学里已经有风险指标 + Kelly 直觉，但作为运营人，你需要把它**制度化**，而不仅是会算：

- **账户层级风控规则**
  - 单日最大亏损限制  
  - 总最大回撤阈值（如 20%、30%）  
  - 触发后的动作：减仓 / 停止交易 / 只平不开
- **策略层级风控规则**
  - 单策略最大回撤  
  - 最大连续亏损次数  
  - 触发停用/降权的条件
- **仓位与杠杆规范**
  - 每个策略的仓位上限  
  - 每个标的/单笔的杠杆上限  
  - “保守/标准/激进”三档模板
- **极端情况预案**
  - 交易所宕机  
  - 暴跌/插针  
  - 单个币暴雷（例如 FTX/UST 类事件）

目标：让你的系统有一份**可执行的风险管理手册**，而不是脑子里的模糊感觉。

> 建议文件名：`docs/guide/风控与资金管理实务清单.md`

---

### 4. 产品与业务/合规思维清单（小型量化私募视角）

这是现在几份文档里**最空白但对“运营人”最关键**的一块：

- **产品结构**
  - 开放式/封闭式产品  
  - 锁定期、申购赎回规则  
  - 收益分成结构（管理费 + 业绩提成/高水位线等）
- **信息披露与报表**
  - 月度/季度报告内容结构  
  - 净值曲线、最大回撤、重大策略调整说明  
  - 回撤期如何和资金方沟通
- **合规与边界意识**（原则层面，而不是具体法律操作）
  - 适当性原则（资金方风险承受能力）  
  - 不承诺保本/保收益  
  - 信息透明、风险提示（至少在文档层面说清楚）
- **商业模式与成本意识**
  - 技术成本、数据成本、交易成本、合规成本  
  - 目标规模 vs 可支持的策略类型（资金太小/太大都有限制）

目标：让你**从一开始就按“产品+业务”的视角设计系统**，而不是只盯着回测曲线。

> 建议文件名：`docs/guide/量化产品与运营思维清单.md`

### 5. 运营与流程管理清单（你是“总设计师”）

作为“小型私募的运营人”，你要管理的是**一整套流程**，而不仅是策略本身：

- **研究与上线流程**
  - 策略 idea 立项 → 设计 → 回测 → 模拟盘 → 小资金试跑 → 正式纳入组合  
  - 每一步的“过关标准”是什么
- **日内/日常运营流程**
  - 每日开盘前 checklist（数据更新情况、系统状态检查）  
  - 盘中监控项（回撤、异常波动、系统错误）  
  - 收盘后复盘内容（当日盈亏、风控触发情况、异常说明）
- **文档与版本管理**
  - 策略说明书  
  - 回测报告模版  
  - 变更日志（何时改了什么参数、为什么改）
- **事故响应流程**
  - 出现重大亏损/技术故障时，谁负责决策、如何记录、如何复盘

目标：让系统从“一堆策略 + 代码”升级为“**一套可持续运转的流程**”。

> 建议文件名：`docs/guide/量化系统运营流程清单.md`

### 6. 心理与决策习惯清单（尤其在回撤期）

这块在文档中基本没展开，但现实中非常关键：

- **风险承受与预期管理**
  - 接受什么级别的最大回撤  
  - 回撤时的“允许自己做/不允许自己做”的行为
- **决策机制**
  - 重大变更（停用策略、降低总仓位）之前要满足哪些客观条件  
  - 用 checklist 辅助决策，而不是情绪
- **复盘与学习机制**
  - 大亏损后的 post-mortem 模板  
  - 把“运气因素”和“决策质量”分开评估
- **沟通心态**
  - 如何向资金方解释回撤、解释策略失效  
  - 坦诚 + 数据支撑，而不是讲故事

目标：让你在压力下仍然能按**事先设计好的机制做理性决策**。

> 建议文件名：`docs/guide/交易心理与决策习惯清单.md`
