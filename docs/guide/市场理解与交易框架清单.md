# 市场理解与交易框架清单

---

## 一、使用说明

- **目标**：
  - 帮你在设计 / 评估任何策略前，先搞清楚：
    - 这是一个什么样的市场？
    - 你的策略在什么结构下赚钱？
    - 你大致在“对谁的错误”赚钱？
- **使用方式**：
  - 不需要一次性全部掌握；
  - 每次做策略 / 调整仓位前，用这份清单快速过一遍关键问题；
  - 有新认知就往里增补例子和坑位。

## 二、整体市场认知

### 2.1 市场参与者地图【必】

- 你要能列出并大致理解：
  - **个人投机者 / 散户**：
    - 典型行为：追涨杀跌、情绪化交易、高杠杆梭哈；
    - 优势：反应快；
    - 弱点：纪律差、容易被情绪和新闻裹挟。
  - **中长期投资者 / 资金方**：
    - 典型行为：关注大趋势、项目基本面；
    - 优势：资金体量较大、持有周期长；
    - 弱点：入场 / 离场节奏容易受宏观情绪影响。
  - **专业量化 / 做市商 / 机构**：
    - 典型行为：高频交易、做市、统计套利；
    - 优势：技术和执行能力强、成本低；
    - 弱点：有制度约束、风控红线、规模过大时也有“笨重”问题。
  - **项目方 / 基金会 / 早期投资人**：
    - 典型行为：解锁、抛压、回购、市值管理；
    - 他们的行为经常塑造中长期供给格局。

> 自检问题：我的策略主要是站在谁的对立面？是在利用散户的情绪，还是利用机构执行上的限制？

### 2.2 Crypto 市场的特殊性【必】

- **7x24 小时交易**：没有统一“收盘价”，回测和风控要自己定义时间粒度。  
- **高波动 & 高频极端事件**：插针、闪崩、单币暴雷都比传统市场更常见。  
- **情绪与叙事驱动更强**：热点轮动、meme、宏观 / 监管消息的影响很大。  
- **链上透明度**：部分行为（大额转账、合约交互）是可见的，可以作为信号。  
- **监管与合规不确定性**：不同地区监管差异大，政策事件可能触发巨幅波动。

> 自检问题：我在做回测 / 风控时，有没有显式考虑“7x24 + 高波动 + 极端事件更频繁”这几点？

## 三、交易标的与产品结构

### 3.1 标的分类【必】

- **主流币**（BTC、ETH 等）：
  - 流动性好、机构参与度高；
  - 边际信息更多来自宏观、衍生品市场。
- **大市值 Alt**：
  - 有一定叙事和生态，波动高于主流币；
  - 更容易受赛道轮动影响（L2、DeFi、AI 等）。
- **小币 / Meme**：
  - 深度差、情绪主导、容易暴涨暴跌；
  - 执行风险（滑点、冲击成本）很高。
- **稳定币**：
  - 主要关注点是“是否真的稳定”和相关利率 / 收益机会。

> 自检问题：我的策略假设在主流币上成立，换到小币是否仍然合理？流动性风险会不会完全改变策略表现？

### 3.2 交易产品结构【必 + 进阶】

- **现货**【必】：
  - 直接持有资产，无爆仓风险，但会承受完整价格波动。
- **永续合约**【必】：
  - 没有到期日；
  - 通过 **资金费率** 来让合约价格围绕现货价格震荡。
- **季度 / 交割合约**【进阶】：
  - 有明确到期日；
  - 存在价差交易机会（期现套利等）。
- **期权**【进阶】：
  - 看涨 / 看跌期权，适合做波动率交易和保护性策略。

> 自检问题：当前策略是否隐含使用杠杆？是否依赖资金费率 / 基差（期货-现货价差）？

## 四、交易机制与成本

### 4.1 订单与撮合机制【必】

- **订单类型**：
  - 限价单（Limit）：指定价格挂单；  
  - 市价单（Market）：按当前可成交价格立即成交；  
  - 部分平台还有止损单、条件单等。
- **挂单 / 吃单（Maker / Taker）**：
  - 挂单提供流动性，通常手续费更低（甚至为负）；
  - 吃单消耗流动性，手续费更高且容易产生滑点。
- **撮合原则**：
  - 一般遵循价格优先，其次时间优先。

> 自检问题：我的策略主要是挂单还是吃单？如果大量吃单，会不会把回测中的 edge 全贡献给手续费和滑点？

### 4.2 成本结构：手续费、点差、滑点【必】

- **手续费（Fee）**：
  - 按成交金额的一定比例收取；
  - Maker / Taker 费率不同。
- **点差（Spread）**：
  - 买一和卖一之间的价差；
  - 小币、冷门时间段点差会显著变大。
- **滑点（Slippage）**：
  - 实际成交价格与下单预期价格之间的差异；
  - 来源：自身单量太大、市场瞬间波动、深度不足。

> 自检问题：
> - 回测里有没有考虑手续费和简单滑点？  
> - 在真实盘口，用当前深度模拟下单，策略是否依然有边际收益？

### 4.3 杠杆、保证金与强平【必】

- **杠杆（Leverage）**：
  - 用更少保证金放大头寸；
  - 实际上同时放大盈亏与爆仓风险。
- **保证金（Margin）**：
  - 维持仓位所需的最低资金；
  - 跌破维持保证金会触发强平。
- **强平机制**：
  - 不同交易所有不同公式；
  - 本质是：价格到达某个点后，系统强制平仓以防穿仓。

> 自检问题：策略在极端行情下，是否可能短时间触发强平？历史最大回撤 + 杠杆倍数是否在可承受范围内？

### 4.4 资金费率与持仓成本【必】

- **资金费率（Funding Rate）**：
  - 永续合约中，多空双方定期互相支付；
  - 正资金费率：多头付空头；负资金费率：空头付多头。
- **长期持仓的成本 / 收益**：
  - 高正资金费率时长期做多成本很高；
  - 高负资金费率时长期做空成本很高。

> 自检问题：策略是否对资金费率敏感？在资金费率极端时，这个策略是否还适合持有？

## 五、市场状态（Regime）与行情分类

### 5.1 行情分类【必】

- **趋势行情**：
  - 价格单边上涨或下跌，中间回调有限；
  - 趋势跟随策略表现较好，均值回归型策略容易连续止损。
- **震荡行情**：
  - 价格在区间内来回波动，没有明确方向；
  - 均值回归策略表现较好，追趋势策略容易假突破。
- **高波动 / 极端行情**：
  - 短时间内大幅上下扫；
  - 止损触发频繁、滑点巨大、强平风险上升。

### 5.2 波动率与成交量信号【进阶】

- **波动率 Regime**：
  - 低波动：容易横盘 / 盘整；
  - 高波动：容易出现趋势 / 极端事件。
- **成交量结构**：
  - 放量上涨、放量下跌、缩量横盘，各自含义不同。

> 自检问题：我的策略更适合哪种 Regime？有没有简单指标帮助事先识别“不适合当前环境”并自动减仓 / 休息？

## 六、典型“别人输钱的方式”清单

> 目标：知道市场里常见的错误模式，帮助你理解“你在赚谁的钱”。

- **高杠杆梭哈单边**：
  - 小波动就可能被强平；
  - 通常出现在情绪极端乐观时。
- **追涨杀跌 / 情绪化交易**：
  - 上涨时 FOMO 追高，回调时恐慌砍在地板；
  - 高频切换方向导致手续费 + 滑点吞噬资金。
- **不设止损 / 无限补仓**：
  - 坚信“总会回来的”；
  - 在极端行情中一路补到爆仓或项目死亡。
- **全仓押注单一热点 / 小币**：
  - 赛道轮动时容易被“换题材”抛弃；
  - 小币流动性差、做大单很难出场。
- **盲从网红策略 / 号召**：
  - 不了解逻辑，只看过去曲线；
  - 策略失效时，自己不知道该不该停。

> 自检问题：
> - 我的策略在什么情况下，是在收割这些典型错误？  
> - 有没有可能我自己就是在做这些错误行为的“升级版”？

## 七、把市场理解用到策略上的小框架

> 每当你设计一个新策略，尽量用下面几个问题把“市场理解”写下来。

- **问题 1：这个策略在什么市场结构下最赚钱？**
  - 趋势市 / 震荡市 / 极端事件？
  - 高波动 / 低波动？
  - 流动性好还是一般？

- **问题 2：这个策略主要在“谁输钱的时候”赚钱？**
  - 追涨杀跌的散户？
  - 被动平仓 / 强平的高杠杆玩家？
  - 需要对冲 / 再平衡头寸的大资金？

- **问题 3：执行层面的主要风险是什么？**
  - 滑点、点差、手续费会不会吃掉全部 edge？
  - 小币流动性不足，能不能承载策略规模？
  - 极端行情下，是否会被强平或被迫砍在地板？

- **问题 4：当前真实市场环境，接近哪种 Regime？**
  - 与策略最适合的 Regime 是否匹配？
  - 如果不匹配，是否应该减仓、暂停或者切换策略？

## 八、建议的学习顺序

1. 先搞清楚 **参与者地图 + Crypto 特殊性**；  
2. 再理解 **具体交易产品与成本结构**（现货 / 永续 / 杠杆 / 资金费率）；  
3. 然后学会用 **Regime + 别人输钱方式** 去解读历史行情；  
4. 最后把这些内容写进你每一个策略的说明书里，用“谁在输钱、我赚什么钱”的视角审核策略逻辑。
